# Alembic Rules

## Structure
- Store database migration scripts in the `versions` folder.
- Follow the default structure generated by the Alembic CLI.
- Keep `env.py` configured to connect to your SQLAlchemy models.

## Best Practices
- Run `alembic revision --autogenerate -m "message"` to create migrations.
- Always carefully review auto-generated migration scripts before applying them, as Alembic may miss complex changes (like table renames).
- Ensure all model metadata is correctly imported in `env.py`.
- Test both `upgrade` and `downgrade` paths locally to guarantee reversible migrations.
